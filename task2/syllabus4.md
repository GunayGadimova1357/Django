# Декораторы в Python

**Декораторы (Decorators)** — это функции, которые **принимают другую функцию**, расширяют её поведение **без изменения исходного кода** и **возвращают новую функцию**.

Они позволяют добавлять дополнительную логику:

* перед выполнением функции
* после выполнения функции

Декораторы применяются с помощью специального синтаксиса **`@`**.

---

# Основные концепции декораторов

## 1. Функции высшего порядка

Декораторы являются **функциями высшего порядка**, потому что они:

* принимают функцию как аргумент
* возвращают новую функцию

---

## 2. Замыкания

Внутри декоратора обычно используется **вложенная функция (`wrapper`)**, которая:

* вызывает оригинальную функцию
* добавляет дополнительную логику

Такая вложенная функция образует **замыкание**, так как она "помнит" оригинальную функцию.

---

## 3. Синтаксис `@`

Декоратор применяется перед определением функции.

```python id="3d3j6o"
@decorator_name
def function():
    pass
```

Это эквивалентно:

```python id="e3g50h"
function = decorator_name(function)
```

---

# Простой пример декоратора

```python id="uyt6m0"
def my_decorator(func):

    def wrapper():
        print("— До выполнения функции —")
        func()
        print("— После выполнения функции —")

    return wrapper
```

### Использование декоратора

```python id="soxyut"
@my_decorator
def say_hello():
    print("Привет!")
```

### Вызов функции

```python id="39q1c0"
say_hello()
```

### Результат

```
— До выполнения функции —
Привет!
— После выполнения функции —
```

---

# Как работает декоратор

1. Python читает `@my_decorator`
2. Функция `say_hello` передается в `my_decorator`
3. `my_decorator` возвращает функцию `wrapper`
4. Теперь `say_hello` указывает на `wrapper`
5. При вызове `say_hello()` выполняется `wrapper`

---

# Декоратор с аргументами функции

Чаще всего функции принимают аргументы.
Чтобы декоратор работал с ними, используется `*args` и `**kwargs`.

```python id="8rmry9"
def my_decorator(func):

    def wrapper(*args, **kwargs):
        print("Функция вызывается")
        result = func(*args, **kwargs)
        print("Функция завершена")
        return result

    return wrapper
```

# Зачем использовать декораторы

## Логирование

Можно записывать:

* аргументы функции
* время выполнения
* результаты

---

## Проверка прав доступа

Перед выполнением функции можно проверить:

* авторизацию пользователя
* уровень доступа

---

## Замер времени выполнения

Декоратор может измерять **время выполнения функции**.

---

## Кеширование (мемоизация)

Результаты функции можно **сохранять**, чтобы ускорить повторные вызовы.

---

# Встроенные декораторы Python

Python содержит несколько встроенных декораторов.

## `@classmethod`

Метод работает **с классом**, а не с объектом.

```python id="s4b92v"
class Example:

    @classmethod
    def method(cls):
        print(cls)
```

---

## `@staticmethod`

Метод **не использует ни объект, ни класс**.

```python id="v00hpl"
class Example:

    @staticmethod
    def method():
        print("Static method")
```

---

## `@property`

Позволяет обращаться к методу **как к атрибуту**.

```python id="y2x9gk"
class Person:

    def __init__(self, name):
        self._name = name

    @property
    def name(self):
        return self._name
```

Использование:

```python id="9t0rrf"
p = Person("Alex")
print(p.name)
```

---

# Краткий итог

Декораторы — это функции, которые:

* принимают **другую функцию**
* **расширяют её поведение**
* возвращают **новую функцию**
* применяются через **синтаксис `@`**

Они используются для:

* логирования
* проверки доступа
* кеширования
* измерения времени выполнения
* модификации поведения функций без изменения их кода.
