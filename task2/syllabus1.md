# Чистые функции в Python

**Чистые функции (Pure Functions)** — это функции, которые:

* возвращают **одинаковый результат при одинаковых входных данных**
* **не имеют побочных эффектов**

Такие функции зависят **только от переданных аргументов**.
Это делает код проще для **тестирования, отладки и использования принципов функционального программирования**.

---

# Основные характеристики чистых функций

## 1. Детерминированность (Determinism)

Если вызвать функцию несколько раз с одинаковыми аргументами, результат **всегда будет одинаковым**.

### Пример

```python
add_pure(2, 3)  # всегда возвращает 5
```

Это означает, что результат **не зависит от времени, внешних переменных или состояния программы**.

---

## 2. Отсутствие побочных эффектов

Чистая функция **не изменяет внешнее состояние программы**.

Она:

* не изменяет **глобальные переменные**
* не изменяет **мутабельные объекты**, переданные как аргументы (списки, словари)
* не записывает данные в **файлы**
* не взаимодействует с **базами данных**
* не выводит данные в **консоль**
* не зависит от **внешнего состояния**

---

# Пример чистой функции

```python
def add_pure(a, b):
    return a + b
```

### Почему это чистая функция

* результат зависит **только от аргументов**
* нет изменения внешних переменных
* нет побочных эффектов
* одинаковые входные данные → одинаковый результат

---

# Пример нечистой функции (с побочными эффектами)

```python
total = 0

def add_impure(a):
    global total
    total += a  # побочный эффект: изменение глобальной переменной
    return total
```

### Почему это нечистая функция

* изменяет **глобальную переменную**
* результат зависит от **предыдущих вызовов**
* поведение функции **непредсказуемо**

---

# Сравнение чистой и нечистой функции

| Чистая функция                         | Нечистая функция                        |
| -------------------------------------- | --------------------------------------- |
| Нет побочных эффектов                  | Есть побочные эффекты                   |
| Зависит только от аргументов           | Может зависеть от глобального состояния |
| Всегда одинаковый результат            | Результат может меняться                |
| Легко тестировать                      | Тестировать сложнее                     |
| Безопасна для параллельного выполнения | Может вызывать ошибки состояния         |

---

# Преимущества использования чистых функций

## Простота тестирования

Не нужно настраивать состояние программы или окружение.

## Предсказуемость

Одинаковые входные данные всегда дают одинаковый результат.

## Безопасность

Функция не изменяет данные вне себя.

## Легкость понимания

Вся логика находится внутри функции.

## Подходит для функционального программирования

Чистые функции легко комбинировать и использовать в функциональных цепочках.

---

# Краткий итог

Чистая функция:

* зависит только от **аргументов**
* **не изменяет внешние данные**
* **не имеет побочных эффектов**
* **всегда возвращает одинаковый результат**

Такие функции делают код **более надежным, понятным и тестируемым**.
