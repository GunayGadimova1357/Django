# Замыкания (Closures) в Python

**Замыкание (closure)** — это вложенная функция, которая **запоминает и использует переменные из внешней области видимости**, даже после того, как внешняя функция завершила выполнение.

Другими словами, замыкание позволяет функции **"захватывать" своё окружение**.

Это помогает:

* сохранять состояние
* инкапсулировать данные
* избегать использования глобальных переменных

---

# Основные характеристики замыканий

## 1. Вложенная функция

Замыкание всегда создаётся **внутри другой функции**.

```python
def outer():
    def inner():
        pass
```

---

## 2. Свободные переменные

Внутренняя функция использует **переменные из внешней функции**.

Такие переменные называются **свободными переменными (free variables)**.

---

## 3. Возврат функции

Чтобы создать замыкание, внешняя функция **возвращает внутреннюю функцию**.

После этого внутренняя функция продолжает **помнить переменные из внешней области**.

---

# Пример замыкания

```python
def outer_func(text):
    # переменная во внешней области

    def inner_func():
        # внутренняя функция использует переменную text
        print(text)

    return inner_func
```

### Использование

```python
my_func = outer_func("Hello, Closure!")
my_func()
```

### Результат

```
Hello, Closure!
```

Хотя `outer_func` уже завершила выполнение, функция `inner_func` **всё ещё имеет доступ к переменной `text`**.

---

# Как работает замыкание

1. Вызывается внешняя функция.
2. Создаётся внутренняя функция.
3. Внутренняя функция **запоминает переменные внешней функции**.
4. Внешняя функция возвращает внутреннюю функцию.
5. Внутренняя функция может использовать сохранённые переменные.

---

# Изменение переменных внешней области

Чтобы изменить переменную из внешней функции, используется ключевое слово **`nonlocal`**.

### Пример

```python
def counter():
    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment
```

### Использование

```python
c = counter()

print(c())  # 1
print(c())  # 2
print(c())  # 3
```

Функция **сохраняет состояние между вызовами**.

---

# Когда использовать замыкания

## 1. Декораторы

Замыкания лежат в основе **декораторов** в Python.

Они позволяют **оборачивать функции и изменять их поведение**.

---

## 2. Сохранение состояния

Замыкания можно использовать как **лёгкую альтернативу классам**, когда нужно хранить состояние.

---

## 3. Создание специализированных функций

Можно создавать функции на основе **общих шаблонов**.

### Пример

```python
def power_of(n):
    def power(x):
        return x ** n
    return power
```

### Использование

```python
square = power_of(2)
cube = power_of(3)

print(square(4))  # 16
print(cube(2))    # 8
```

---

# Замыкания vs Классы

| Замыкания                       | Классы                      |
| ------------------------------- | --------------------------- |
| Проще и короче                  | Более структурированы       |
| Хороши для небольшого состояния | Подходят для сложной логики |
| Используют функции              | Используют объекты          |

---

# Краткий итог

Замыкания — это функции, которые:

* создаются **внутри другой функции**
* **запоминают переменные внешней области**
* продолжают использовать их **после завершения внешней функции**
* позволяют **сохранять состояние без глобальных переменных**

Чаще всего замыкания используются для:

* декораторов
* создания специализированных функций
* хранения состояния между вызовами.
