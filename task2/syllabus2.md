# Генераторы в Python

**Генераторы (Generators)** — это специальные функции, которые возвращают **итератор**, генерирующий значения **по одному во время итерации**, а не создающий весь набор данных сразу.

Вместо `return` генераторы используют ключевое слово **`yield`**, которое:

* возвращает значение
* **приостанавливает выполнение функции**
* сохраняет текущее состояние функции

Это позволяет **экономить память**, особенно при работе с **большими данными** (например, при чтении больших файлов).

---

# Основные понятия

## `yield`

Ключевое слово `yield`:

* возвращает значение из функции
* **останавливает выполнение функции**
* сохраняет состояние функции

При следующем вызове генератор **продолжает выполнение с того места**, где остановился.

---

## Итератор

Генератор возвращает **объект-итератор**, который можно использовать:

* в цикле `for`
* через функцию `next()`

---

## Ленивые вычисления (Lazy Evaluation)

Генераторы **не создают весь набор данных сразу**.

Они:

* создают элементы **по мере необходимости**
* используют **минимум памяти**

Это делает их идеальными для:

* больших наборов данных
* потоковых данных
* бесконечных последовательностей

---

# Способы создания генераторов

## 1. Генераторная функция (через `yield`)

```python
def count_up_to(n):
    i = 1
    while i <= n:
        yield i
        i += 1
```

### Использование

```python
for num in count_up_to(3):
    print(num)
```

### Результат

```
1
2
3
```

Каждое значение создаётся **только тогда, когда оно требуется циклу**.

---

## 2. Генераторные выражения

Генераторные выражения похожи на **list comprehension**, но используют **круглые скобки**.

```python
squares = (x**2 for x in range(5))
```

### Использование

```python
print(next(squares))  # 0
print(next(squares))  # 1
```

Можно преобразовать генератор в список:

```python
print(list(squares))
```

---

# Преимущества генераторов

## Экономия памяти

Генераторы **не хранят все данные в памяти**, а создают элементы по мере необходимости.

---

## Работа с большими данными

Идеально подходят для:

* больших файлов
* больших списков
* потоков данных

---

## Поддержка бесконечных последовательностей

Генераторы могут создавать **бесконечные последовательности**.

Пример:

```python
def infinite_numbers():
    n = 1
    while True:
        yield n
        n += 1
```

---

## Более лаконичный код

Генераторы позволяют писать **короткий и читаемый код**, особенно при обработке данных.

---

# Отличие генератора от обычной функции

| Обычная функция              | Генератор                        |
| ---------------------------- | -------------------------------- |
| Использует `return`          | Использует `yield`               |
| Выполняется сразу            | Выполняется по шагам             |
| Возвращает значение          | Возвращает объект-генератор      |
| Завершается после выполнения | Приостанавливается после `yield` |

---

# Когда использовать генераторы

Генераторы полезны в следующих ситуациях:

* чтение **больших файлов построчно**
* обработка **больших потоков данных**
* создание **ленивых вычислений (lazy evaluation)**
* работа с **бесконечными последовательностями**
* обработка данных **без загрузки их полностью в память**

---

# Краткий итог

Генераторы — это функции, которые:

* используют `yield`
* возвращают **итератор**
* создают значения **по одному**
* экономят **оперативную память**
* подходят для **больших и бесконечных данных**
